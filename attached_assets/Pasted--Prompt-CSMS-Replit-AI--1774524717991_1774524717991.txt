# 🚀 Prompt: نظام إدارة خدمة العملاء (CSMS) — مناسب لـ Replit AI
---
## 🎯 نظرة عامة على المشروع
ابنِ نظامًا إلكترونيًا متكاملًا لإدارة خدمة العملاء (Customer Service Management System - CSMS) باستخدام:
- **Frontend:** React + Next.js + Ant Design (دعم RTL للعربية)
- **Backend:** Node.js + Express (أو NestJS) + Prisma ORM
- **Database:** PostgreSQL
- **Auth:** NextAuth.js (Email/Password + Google OAuth 2.0)
- **Real-time:** Socket.io للإشعارات الفورية
- **Storage:** Cloudinary أو Local Storage للصور
- **Charts:** Recharts أو Chart.js
- **Deploy:** Docker + Docker Compose
- **Theme:** Dark Mode افتراضي بالكامل لجميع الصفحات والمكونات

## 🔐 المصادقة والأمان (Authentication & Security)
### تسجيل الدخول
- بريد إلكتروني + كلمة مرور (JWT Access Token + Refresh Token)
- تسجيل الدخول عبر Google (OAuth 2.0) — يُفعَّل من إعدادات النظام
- Rate Limiting على محاولات الدخول الفاشلة
- تشفير كلمات المرور بـ Bcrypt
### المستخدم الافتراضي (يُنشأ تلقائيًا عند أول تشغيل)
```
username: wael
password: 123
role: مدير (Manager)
```
### بنية User Entity
```
id, name, email, password_hash, google_id, role_id, created_at
```

## 👥 الأدوار والصلاحيات (Roles & Permissions)
| الدور | الصلاحيات |
|-------|-----------|
| **موظف خدمة عملاء** | إضافة/تعديل/متابعة عملاء، إنشاء شكاوى، استلام شكاوى، تحديث حالة، تسجيل حلول |
| **محاسب** | استيراد بيانات العملاء والفواتير، مشاهدة تقارير مالية، استلام شكاوى إدارية |
| **مدير** | كل الصلاحيات: إعدادات النظام، التحكم في الصلاحيات، الموافقة على التصعيد |
| **مدير/وتر** | نفس المدير + إلغاء/تعديل أي سجل، مشاهدة جميع السجلات |
| **مهندس صيانة** | صلاحية شاملة: إدارة الفروع، المنتجات، الموظفين، النظام كله |

## 📋 وظائف النظام التفصيلية (Functional Requirements)
### 1️⃣ تسجيل العملاء
#### يدويًا — نموذج "إضافة عميل جديد":
- كود العميل (اختياري — يُولَّد تلقائيًا بصيغة `CUST-YYYYMMDD-####`)
- اسم العميل* (نص، أقصى 150 حرف)
- رقم الهاتف* (7–15 رقمًا)
- نوع العميل* (قائمة منسدلة من الإعدادات)
- المحافظة* (قائمة منسدلة)
- عنوان الشارع (اختياري)
- الفرع* (من الفروع المعرفة)
#### استيراد CSV:
- المسار: `الإعدادات → إدارة البيانات → استيراد بيانات العملاء`
- خياران للاستيراد:
- استيراد عملاء فقط
- استيراد عملاء + فواتير
- دعم ملفات UTF-8، بحد أقصى 50,000 صف في الدفعة
- أعمدة العميل: `code, name, phone, type, governorate, branch, address`
- أعمدة الفاتورة: `invoice_number, invoice_date, invoice_amount, invoice_status, invoice_product`
#### قواعد الاستيراد:
- التكرار يُحدَّد بـ `name + phone`؛ إذا وُجد يُحدَّث السجل
- تغيير الفرع يُسجَّل في `BranchChangeLog`
- توليد `code` تلقائيًا إذا كان فارغًا
- التحقق من وجود الفرع والمنتج؛ إذا لم يُوجَد تُسجَّل تحذيرات وتستمر العملية
- فحص تنسيق التاريخ (`YYYY-MM-DD` أو `DD/MM/YYYY`)
- التحقق من أن المبلغ رقم عشري مقرَّب لمنزلتين
- جمع الأخطاء والتحذيرات في `ImportLog`
