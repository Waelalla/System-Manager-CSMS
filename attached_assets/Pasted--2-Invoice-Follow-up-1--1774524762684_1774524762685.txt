
### 2️⃣ متابعة الفواتير (Invoice Follow-up)
1. بعد الاستيراد → الانتقال إلى صفحة "متابعة العملاء"
2. زر **"إضافة عملاء اليوم"** → عرض الفواتير المستوردة اليوم غير المتابعة
3. الجدول يحتوي: رقم الفاتورة، التاريخ، اسم العميل، الفرع، حالة المتابعة
4. المستخدم يختار الفواتير → زر **"متابعة مختارة"**
5. نموذج التقييم يُحدِّده المدير من `الإعدادات → أسئلة المتابعة` (نجوم، اختيار، نص)
6. بعد الحفظ → إنشاء سجل `FollowUp`
7. إذا كان التقييم < الحد الأدنى (مثلًا 3 نجوم) → عرض زر **"إنشاء شكوى"** مع تعبئة مسبقة للبيانات

### 3️⃣ إدارة الشكاوى (Complaints)
#### تعريف أنواع الشكاوى — من الإعدادات:
- اسم النوع (مثال: "مشكلة جودة المنتج")
- قائمة الحقول لكل نوع:
- نوع الإدخال: نص قصير، textarea، رقم، تاريخ، قائمة منسدلة، رفع ملف، تقييم نجوم
- إلزامي / اختياري
- خيارات القائمة المنسدلة (إن وُجدت)
#### إنشاء شكوى جديدة:
1. اختيار العميل (بحث بالاسم أو رقم الهاتف)
2. اختيار نوع الشكوى → توليد حقول ديناميكية تلقائيًا
3. الحقول الثابتة المشتركة:
- قناة التواصل (هاتف / واتساب / بريد إلكتروني / ...)
- الأولوية (عادية / عالية)
- وصف المشكلة (textarea)
- إرفاق صور (اختياري)
4. زر **"حفظ"** → إنشاء سجل `Complaint` بحالة **"جديدة"**
#### دورة حياة الشكوى:
| الخطوة | ما يحدث | من يملك الصلاحية |
|--------|---------|-----------------|
| إشعار بالشكوى | Toast + إشعار داخلي فوري | جميع موظفي خدمة العملاء |
| استلام الشكوى | تغيير الحالة إلى "مستلمة" + تعيين `assigned_to_id` | موظف خدمة عملاء |
| قيد الحل | تغيير الحالة + إضافة ملاحظات مبدئية | المستلم |
| حل الشكوى | ملء حقل "حل الشكوى" + تغيير الحالة إلى "محلول" | المستلم |
| نقل الشكوى | طلب تحويل إلى موظف آخر (يقبل أو يرفض) | المستلم |
| تصعيد إداري | إشعار لجميع المدراء والمحاسبين | المستلم |
| تقييم العميل | نافذة تقييم نجوم (1–5) + تعليق → حفظ في `Feedback` | المستلم |
| إغلاق الشكوى | تغيير الحالة إلى "مغلق" + تسجيل كل الإجراءات في `ComplaintLog` | المستلم / المدير |

### 4️⃣ التقارير التحليلية (Analytics)
#### لوحة التحكم — KPIs:
- عدد الشكاوى الجديدة / المستلمة / المغلقة (يومي / أسبوعي / شهري)
- متوسط زمن الحل (دقائق / ساعات)
- عدد الشكاوى المصعَّدة إداريًا
- متوسط تقييم العميل حسب الفروع
- مخططات: خطية (Trend)، شريطية (توزيع حسب النوع)
#### تحليل الشكاوى:
- تصفية حسب النوع، الفرع، المنتج، الأولوية، التاريخ
- مخطط دائري لنسبة كل نوع شكوى
- جدول أعلى 5 منتجات تسبب شكاوى
- متوسط مدة حل كل نوع شكوى (رسم شريطي)
#### تحليل العملاء:
- أعلى 10 عملاء من حيث إجمالي المشتريات
- عدد الشكاوى لكل عميل
- تغيّر الفروع (من `BranchChangeLog`) مع زر "مراجعة"
#### تحليل الاستفسارات:
- عدد الفواتير غير المتابعة
- متوسط زمن المتابعة من تاريخ الفاتورة
- قائمة الفواتير ذات التقييم المنخفض (≤ 2 نجوم)
#### تقارير الفروع:
- لكل فرع: عدد العملاء النشطين، إجمالي الانطباعات، متوسط تقييم المنتج، متوسط تقييم الخدمة، آخر التعليقات
- زر **"اتصل بالعميل"** → نموذج تسجيل مكالمة وملاحظات
- زر **"تغيير فرع"** → عرض عملاء غيّروا فرعهم مؤخرًا

## 📄 وصف كل صفحة في التطبيق (Pages)
### صفحة تسجيل الدخول (Login)
- حقل البريد الإلكتروني + كلمة المرور
- زر "تسجيل الدخول"
- زر "تسجيل الدخول بـ Google"
- رابط "نسيت كلمة المرور؟" (إرسال بريد إعادة تعيين)
- بعد النجاح: توجيه حسب دور المستخدم
### لوحة التحكم (Dashboard)
- بطاقات KPI (شكاوى، فواتير متابعة، تقييمات)
- مخطط زمني لعدد الشكاوى خلال الشهر
- روابط سريعة: إضافة عميل / استيراد / إنشاء شكوى
- الموظف: مهام يومية (فواتير بانتظار متابعة، شكاوى جديدة)
- المدير: تقارير الأداء + إعدادات النظام
### صفحة العملاء (Customers)
 لا تعرض العملاء كلهم في الصفحة علشان لو في 50الف عميل البينات متبقاش جنب بعض بل من الافضل ان يكون في ازرار تنقل بين الصفح مثلا دي الصفحة رقم 1
- قائمة مع فلاتر (اسم، هاتف، فرع، نوع، محافظة)
- زر "إضافة عميل يدوي"
- زر "استيراد CSV"
- أعمدة: كود العميل، الاسم، الهاتف، النوع، الفرع، عدد الفواتير، آخر فاتورة
- إجراءات: إضافة، تعديل، حذف (حسب الصلاحية)
### نموذج إضافة/تعديل عميل
- الحقول: كود، اسم، هاتف، نوع، محافظة، فرع، عنوان
- تعديل الفرع → إضافة سجل في `BranchChangeLog`
### صفحة استيراد البيانات (Import Data)
- اختيار ملف CSV
- خيارات: استيراد عملاء فقط / استيراد عملاء + فواتير / تحديث الموجود
- Progress bar أثناء الاستيراد
- ملخص بعد الانتهاء: مضاف، محدَّث، أخطاء، فواتير مكررة، عملاء غيّروا فرعهم
### صفحة متابعة الفواتير (Invoice Follow-up)
- زر "إضافة عملاء اليوم"
- جدول الفواتير غير المتابعة
- تحديد الفواتير + زر "متابعة مختارة"
- نموذج التقييم (حسب إعدادات المدير)
- عند التقييم المنخفض: زر "إنشاء شكوى" مع تعبئة مسبقة
### صفحة الشكاوى (Complaints)
- قائمة مع فلاتر: الحالة، النوع، الفرع، الموظف، الأولوية، التاريخ
- أعمدة: رقم الشكوى، العميل، النوع، الحالة، الموظف المسؤول، التاريخ
- أزرار السطر: استلام، عرض، تحويل، إغلاق
### نموذج إضافة شكوى جديدة
- خطوة 1: اختيار العميل (بحث)
- خطوة 2: اختيار نوع الشكوى → حقول ديناميكية
- خطوة 3: الحقول الثابتة (قناة، أولوية، وصف، صور)
- حفظ → سجل بحالة "جديدة"
### صفحة تفاصيل الشكوى (Complaint Detail)
- المعلومات الأساسية: رقم، عميل، فرع، فاتورة، نوع، قناة، أولوية، تاريخ
- أزرار الإجراء حسب الحالة: استلام، قيد الحل، نقل، تصعيد، حل، إغلاق
- سجل الأنشطة (ComplaintLog) زمنيًا باسم المستخدم
- قسم تقييم العميل (نجوم + تعليق) — يظهر بعد "تم الحل"
- الملفات المرفقة (صور، مستندات)
### صفحة التقارير (Analytics / Reports)
- تبويبات: ملخص الأداء / تحليل الشكاوى / تحليل العملاء / تحليل الاستفسارات / تقارير الفروع
- تصدير كل تبويب إلى CSV أو Excel
### صفحة الإعدادات (Settings) — قائمة جانبية
1. أنواع الشكاوى — إدارة الحقول الديناميكية
2. الفروع — إضافة / تعديل / حذف فروع
3. المنتجات — إدارة المنتجات والخصائص
4. الموظفون والصلاحيات — إنشاء / تعديل / حذف + تعيين أدوار
5. معلومات الشركة — اسم، شعار، عنوان، رقم
6. إعدادات البريد — تكامل EmailJS (API Key + Template)
7. تصنيف العملاء — قواعد النقاط والمستويات
8. نظام النقاط — كسب وإنفاق النقاط
9. المظهر — ألوان، شعار، Dark/Light Mode
10. Google OAuth — Client ID و Secret وتفعيل/تعطيل
11. استيراد البيانات — سلوك التكرار، الدفعات، تنسيق التاريخ
12. الأمان — سياسات كلمة المرور، جلسات، 2FA
### صفحة سجل تغيير الفروع (Branch Change Log)
- جدول: كود العميل، اسم، هاتف، الفرع القديم، الفرع الجديد، التاريخ
- زر "اتصل" → Modal لتسجيل ملاحظات المكالمة
### صفحة سجلات الاستيراد (Import Log)
- جدول عمليات الاستيراد السابقة مع الإحصائيات
- زر "عرض التفاصيل" → قائمة الأخطاء (السطر، العمود، السبب)
### الملف الشخصي (Profile)
- اسم المستخدم، البريد، الدور
- تعديل كلمة المرور
- ربط / إلغاء ربط حساب Google
### الإشعارات (Notifications)
- قائمة منسدلة في الشريط العلوي
- شكاوى جديدة، طلبات تحويل، تصعيد إداري
- النقر → توجيه للصفحة / المهمة المعنية
